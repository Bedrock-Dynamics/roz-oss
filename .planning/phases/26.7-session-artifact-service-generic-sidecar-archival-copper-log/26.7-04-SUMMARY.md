## Figment Env-Var Path (Operator Reference)

Workers set per-session copper recording policy via either TOML or env vars; env vars override TOML (figment merge precedence unchanged):

**TOML (`roz-worker.toml`):**
```toml
[observability.copper]
preallocated_mb = 256
keep_local_after_upload = false
```

**Environment variables (double-underscore separator between nested keys):**
```
ROZ_OBSERVABILITY__COPPER__PREALLOCATED_MB=256
ROZ_OBSERVABILITY__COPPER__KEEP_LOCAL_AFTER_UPLOAD=false
```

Single-underscore keys (e.g. `ROZ_API_URL`, `ROZ_MAX_CONCURRENT_TASKS`) are untouched — figment only splits on the literal `__` separator, and no existing flat struct field contains that substring.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Added `.split("__")` to `WorkerConfig::load`'s `Env::prefixed(...)` layer**
- **Found during:** Task 2 RED — new `config_observability_loads_nested_copper_overrides` test set `ROZ_OBSERVABILITY__COPPER__PREALLOCATED_MB=64` via `figment::Jail`, ran `WorkerConfig::load()`, and asserted `cfg.observability.copper.preallocated_mb == 64`; actual value stayed at the default 256.
- **Root cause:** `figment::providers::Env::prefixed("ROZ_")` does not split nested keys by default. Figment requires an explicit `.split("__")` to treat `__` as a path separator (figment-0.10.19 `src/providers/env.rs:409`). The Phase 26.5 module-level docs on `observability_config.rs` documented `ROZ_OBSERVABILITY__CAMERA__RECORD` as supported, but the existing `config_observability_loads_nested_overrides` test exercised the nested path via `Serialized::defaults(json!({"observability": {...}}))`, not env vars — so the gap was documented-but-untested.
- **Fix:** Added `.split("__")` to the `Env::prefixed("ROZ_")` layer in `WorkerConfig::load`. Single-underscore flat keys (`ROZ_API_URL`, `ROZ_MAX_CONCURRENT_TASKS`, `ROZ_WORKER_ID`, `ROZ_DATA_DIR`, `ROZ_MODEL_TIMEOUT_SECS`, etc.) are unaffected because none contain the literal `__` substring. Inline comment at the call site explains the scope.
- **Files modified:** `crates/roz-worker/src/config.rs`
- **Commit:** `48460c1`
- **Incidental benefit:** The previously-broken `ROZ_OBSERVABILITY__CAMERA__*` env-var surface now also flows through end-to-end. No test was added for the camera side — scope-capped to what the plan required.

**2. [Rule 3 - Blocking] Added `figment` dev-dep with `test` feature**
- **Found during:** Task 2 RED compile — `figment::Jail` is gated behind the `test` feature upstream (compile error E0433 "could not find `Jail` in `figment`"; figment-0.10.19 `src/lib.rs:594` `#[cfg(any(test, feature = "test"))]`).
- **Fix:** Added `figment = { workspace = true, features = ["env", "toml", "test"] }` to `roz-worker`'s `[dev-dependencies]` section. Runtime builds remain unchanged (workspace dep stays at `["env", "toml"]`).
- **Files modified:** `crates/roz-worker/Cargo.toml`
