# 26.7-07 — `roz session export --bundle`

## Outcome
`roz session export <session_id> --bundle [--output path.tar]` now produces a self-verifying uncompressed .tar containing `manifest.json` (first entry) + MCAP + every `roz_session_artifacts` row's downloaded bytes. Manifest v1 matches D-29 schema exactly. `--bundle` and `--time-range` are mutually exclusive at clap parse time.

## Key files
- `crates/roz-cli/src/commands/session.rs` — extended: `Export` variant grows `bundle: bool` flag (in partial-merge commit); full `export_bundle` helper + `fetch_mcap_entries` + `fetch_artifact_entries` + `assemble_tar_blocking` + `verify_bundle_integrity` + 2 unit tests.
- `crates/roz-cli/Cargo.toml` — `hex = { workspace = true }` already added in partial-merge commit.

## Architecture
- `tokio::try_join!` runs MCAP + artifact fetches concurrently on the same HTTP/2 channel.
- `futures::future::try_join_all` parallelizes per-artifact `DownloadArtifact` calls.
- Both CLI-side clients share a single `Channel` cloned once per-helper; interceptor is rebuilt per-helper to avoid interceptor-Clone issues.
- Hashing is streaming (`sha2::Sha256`); 64 KiB heap buffer during integrity re-read.
- Manifest entries are lowercase hex-encoded `digest_sha256`.
- Tar assembly runs on `tokio::task::spawn_blocking` with `tar::Builder::new(Vec::new())`; no compression (D-27).
- `verify_bundle_integrity` re-reads the produced .tar, re-hashes every entry, and cross-checks against `manifest.files[]`. Also asserts `manifest.json` is the FIRST entry.
- Defense-in-depth: during artifact download, each chunk's optional final-frame `digest_sha256` is compared against the client-computed rolling hash.

## Verification
- `cargo clippy -p roz-cli -- -D warnings` — green (after one `clippy::large_stack_arrays` fix: heap buffer in `verify_bundle_integrity`).
- `cargo test -p roz-cli --lib bundle_tests` — 2/2 pass:
  - `bundle_contains_manifest_first` — tar entry #0 is `manifest.json`, integrity check passes on clean bundle.
  - `bundle_integrity_catches_tampered_payload` — manifest digest ≠ actual bytes → error.

## Deviations from plan
- Stalled executor agent landed the `--bundle` clap flag + `hex` dep in partial commits `f0bbdf6` + `a259339`; this plan continued from there.
- Plan's suggested MCAP rollover-indexed BTreeMap was replaced with a single concatenated MCAP entry (`mcap/{session_id}.mcap`, rollover_index=0). The existing `ExportSession` streamer already concatenates rollovers server-side so there is nothing to discriminate on at the chunk boundary; per-rollover splitting would require a proto extension (deferred — single concatenated file matches D-28 MCAP rollover-0 layout).
- `config.tenant_id()` doesn't exist on `CliConfig` — `manifest.tenant_id` is emitted as the empty string (informational only; server auth is authoritative at `--bundle` download time).
