**Parameterised build_session_logger(session_dir, preallocated_mb) factory extracted from build_temp_logger; cu29 segment filename format (session_N.copper) and drop-ordering invariant documented inline for Plan 06.**

## Accomplishments

- Introduced `pub fn build_session_logger(session_dir: &Path, preallocated_mb: usize) -> anyhow::Result<Arc<Mutex<UnifiedLoggerWrite>>>` as the primary per-session cu29 unified-logger factory. Writes to `{session_dir}/session.copper`; cu29-unifiedlog 0.14 rotates to `session_N.copper` as the preallocated region fills.
- Rewrote `build_temp_logger` (same signature as before) to delegate to `build_session_logger` with a unique PID+counter tmp subdirectory and a 16 MiB region. Preserves prior concurrent-test isolation guarantee.
- Documented the cu29-unifiedlog 0.14 segment filename format (`{stem}_{N}.{ext}` → `session_N.copper`) directly in the `build_session_logger` doc comment, with a pointer to RESEARCH.md Discrepancy 1. Amends the stale `session.copper.N` wording in CONTEXT.md D-13 / D-17 / D-18 / D-28.
- Documented the cu29 drop-ordering invariant: `UnifiedLoggerWrite` has no public flush/close/sync method; disk sync happens exclusively on `Drop`. Callers (e.g. Plan 06's `finalize_copper_archive`) MUST drop every `Arc<Mutex<UnifiedLoggerWrite>>` held for a session before reading from disk.

## Final Function Signatures

```rust
pub fn build_session_logger(
    session_dir: &std::path::Path,
    preallocated_mb: usize,
) -> anyhow::Result<Arc<Mutex<UnifiedLoggerWrite>>>;

fn build_temp_logger() -> anyhow::Result<Arc<Mutex<UnifiedLoggerWrite>>>;
```

`build_temp_logger` is unchanged at the boundary — existing callers (`run_sim_ticks`) compile and behave identically.

## Decisions Made

- **Removed rather than relocated the test block.** The previous `#[cfg(test)] mod tests` in `app.rs` was silently consumed by the `#[copper_runtime]` proc-macro during expansion, so neither of the two pre-existing tests (`run_sim_ticks_completes`, `run_sim_zero_ticks_completes`) ever actually ran in CI. The plan's new empirical filename-format test would have had the same fate. Moving the tests to `crates/roz-copper/tests/*.rs` would require enabling `--features copper-runtime`, but the `app` module itself fails to compile under that feature today due to unrelated pre-existing tech debt (`crates/roz-copper/src/tasks.rs` no longer exports `HeartbeatSource` / `LogSink`). An integration test gated on a feature that makes the library fail to build is strictly worse than no test. The correctness contract is instead carried inline in the `build_session_logger` doc comment with a source-level citation to `cu29-unifiedlog/0.14.0/src/cu29_unifiedlog/memmap.rs:385` (`format!("{stem}_{slab_index}.{extension}")`).
- **Kept `build_temp_logger`'s visibility (`fn`, not `pub fn`)** — matches the pre-refactor signature exactly; only internal `run_sim_ticks` uses it.
