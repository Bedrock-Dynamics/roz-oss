# Phase 26.7 Plan 02: Proto Contract + Worker Client Codegen Summary

One-liner: Added `ArtifactService` (UploadArtifact client-stream, DownloadArtifact server-stream, ListSessionArtifacts unary) + 9 supporting messages to `observability.proto`, and flipped `roz-worker/build.rs` to `build_client(true)` so the worker can dial `UploadArtifact`.

## What Changed

### `proto/roz/v1/observability.proto` (commit 36c245f)

Appended after the existing `ObservabilityService` — no existing message/service/field-number was touched. New messages with field numbers:

| Message | Fields (tag) |
|---------|--------------|
| `UploadArtifactRequest` | oneof payload: `metadata` (1) \| `chunk` (2) |
| `UploadArtifactMetadata` | session_id (1), artifact_type (2), path (3), size_bytes (4), digest_sha256 (5), content_type (6) |
| `UploadArtifactChunk` | data (1) |
| `UploadArtifactResponse` | artifact_id (1), size_bytes (2) |
| `DownloadArtifactRequest` | artifact_id (1) |
| `DownloadArtifactChunk` | data (1), optional digest_sha256 (2) |
| `ListSessionArtifactsRequest` | session_id (1) |
| `ListSessionArtifactsResponse` | repeated artifacts (1) |
| `ArtifactSummary` | artifact_id (1), artifact_type (2), path (3), digest_sha256 (4), size_bytes (5), content_type (6) |

Service (new — appended at EOF):

```proto
service ArtifactService {
  rpc UploadArtifact       (stream UploadArtifactRequest)   returns (UploadArtifactResponse);
  rpc DownloadArtifact     (DownloadArtifactRequest)        returns (stream DownloadArtifactChunk);
  rpc ListSessionArtifacts (ListSessionArtifactsRequest)    returns (ListSessionArtifactsResponse);
}
```

### `crates/roz-worker/build.rs` (commit 6ccb66f)

Three changes vs prior state:

1. Module-level doc updated to mention Phase 26.7 + ArtifactService client
2. `.build_client(false)` → `.build_client(true)` — enables `ArtifactServiceClient` codegen
3. `compile_protos` list extended with `"../../proto/roz/v1/observability.proto"`

`build_server(false)` retained (worker is gRPC client-only).
