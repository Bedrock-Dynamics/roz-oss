## Success Criterion Verification

### SC1 ‚Äî MAVLink log protocol state machine ‚Äî **PASS**

`crates/roz-mavlink/src/log_download.rs` implements the full protocol:
- `LogDownloader::fetch_newest()` issues `LOG_REQUEST_LIST`, collects `LOG_ENTRY` responses, selects newest by `time_utc` (with `id` tiebreaker per D-02), requests chunked `LOG_REQUEST_DATA`, reassembles `LOG_DATA` with rolling buffer + gap-retry (3 attempts per D-01), sends `LOG_REQUEST_END`.
- **Critical invariant (RESEARCH pitfall 1):** `impl Drop for LogDownloader` at `crates/roz-mavlink/src/log_download.rs:418` best-effort `try_send`s `LOG_REQUEST_END` on every exit path (success, timeout, abort, panic). Verified by test `drop_without_end_sent_issues_log_request_end` in Plan 02 SUMMARY.
- **Safety cap:** `MAX_LOG_SIZE_BYTES = 500 MiB` sanity limit enforced before `Vec` allocation prevents OOM from malicious FC reporting oversized `LOG_ENTRY.size`.
- `AutopilotKind` enum + `from_mavlink_autopilot(u8) -> Self` defined in `crates/roz-mavlink/src/lib.rs` (Plan 02 Task 2) with 4-variant unit test coverage.

### SC2 ‚Äî `[ulog]` config section ‚Äî **PASS**
