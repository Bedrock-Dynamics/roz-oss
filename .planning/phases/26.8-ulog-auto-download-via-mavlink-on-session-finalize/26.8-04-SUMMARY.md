## Decisions Made

- **Signature accepts non-Option `Arc<MavlinkBackend>`.** The `Option<MavlinkBackend>::is_none()` gate lives at the Plan 06 session_relay spawn call-site. Matches the plan's `must_haves.truths`.
- **Return type is `anyhow::Result<Option<String>>`.** Outer `Err` is reserved for future plumbing failures; every known soft-fail path returns `Ok(None)`. `Some(artifact_id)` is Plan 07's LOG_ERASE gate condition.
- **Single `match e { ... }` that binds `failure_mode` then logs once.** This is a minor diff from the plan's inline-per-variant `tracing::warn!` sketch — reduces 6 warn-log sites to 1, keeps the taxonomy table co-located with the dispatch. Still satisfies D-09 (every variant emits a distinct `failure_mode` string).
- **`sha256_of_bytes` is non-streaming.** The copper_archive streams because it reads a file; here the bytes arrive pre-loaded from `LogDownloader::fetch_newest(...)`, so a 5-line `Sha256::new().update(bytes).finalize()` is sufficient. Defer any trait-based dedup to a later phase per the plan's explicit instruction.
