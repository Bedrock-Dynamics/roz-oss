## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 — Bug] Rust 2024 edition match ergonomics rejected explicit `ref` binding**

- **Found during:** Task 1 build step (`cargo build -p roz-worker`).
- **Issue:** The plan's prescribed gate pattern `if let (Some(ref client), Some(ref backend)) = (&artifact_client, &mavlink_backend) { ... }` fails to compile on Rust 2024 (toolchain 1.92.0):
  ```
  error: cannot explicitly borrow within an implicitly-borrowing pattern
    --> crates/roz-worker/src/session_relay.rs:1318:18
     |
  1318 |     if let (Some(ref client), Some(ref backend)) = (&artifact_client, &mavlink_backend) {
     |                  ^^^               ^^^ explicit `ref` binding modifier not allowed when implicitly borrowing
  ```
  Under 2024 edition match ergonomics, destructuring a tuple of references already implicitly borrows the inner `Option<T>` contents; an explicit `ref` is redundant and forbidden.
- **Fix:** Applied the compiler-suggested form: `if let (Some(client), Some(backend)) = (&artifact_client, &mavlink_backend)`. The bindings `client: &ArtifactServiceClient<Channel>` and `backend: &Arc<MavlinkBackend>` then feed `client.clone()` and `std::sync::Arc::clone(backend)` identically to the copper_archive block directly above.
- **Files modified:** `crates/roz-worker/src/session_relay.rs` (1 line).
- **Commit:** `df9f8c9` (folded into the Task 1 commit — caught before the initial `git add`).
- **Note:** Plan 06's acceptance-criteria grep for the literal `if let (Some(ref client), Some(ref backend)) = (&artifact_client, &mavlink_backend)` string is NOT satisfied verbatim; the semantically-equivalent `if let (Some(client), Some(backend)) = (&artifact_client, &mavlink_backend)` is present instead. All other grep criteria pass as written.
