## Decisions Made

- **Fixture sourcing:** D-12 option (c) synthetic generation. PX4 SITL not available in the worktree environment; option (c) has no license constraints and no external fetch, making it the safest and most reproducible path. The orchestrator's `<fixture_sourcing_guidance>` block pre-resolved this, so Task 1's `checkpoint:human-action` was auto-satisfied without pausing the wave.
- **Flight Review UAT (SC4):** Explicitly deferred. A synthetic fixture lacks real flight telemetry, so it will not render as a full flight trace in Flight Review. Plan 03's automated acceptance criteria validate structure (magic header, digest pin, size band). The SC4 UAT target stays open until Phase 27 brings a live SITL fixture.
- **Test layout choice:** Plan `<action>` option (b) — tests live in a dedicated `crates/roz-mavlink/tests/mock_log_transport_unit.rs` file that declares `mod common;`. This is cleaner than inline `#[cfg(test)] mod tests` in the helper and matches the pattern Plan 08's end-to-end test file will extend.
- **Interface expansion:** Added `with_autopilot(AutopilotFamily)` knob not listed in the `<interfaces>` skeleton. Zero default-path cost; Plan 04's `autopilot_kind() != Px4` gating test path is already wired at no additional implementation risk.

**2. [Rule 1 - Bug] Fixed nonsensical `if remainder == 0 { 1 } else { 1 }` in test math**
- **Found during:** Task 2 (clippy pass)
- **Issue:** Initial test code used `full + if remainder == 0 { 1 } else { 1 }` — both branches identical. Clippy `-D warnings` rejected this under `clippy::if-same-then-else`. The math is correct — both branches yield `full + 1` total frames (EOF encoding is either a short final frame or a trailing zero-byte frame) — but the expression was obscured.
- **Fix:** Collapsed to `full + 1` with an inline comment explaining the EOF-encoding invariant.
- **Files modified:** `crates/roz-mavlink/tests/mock_log_transport_unit.rs`
- **Verification:** `cargo clippy -p roz-mavlink --tests -- -D warnings` clean; test still passes.
- **Committed in:** `3406b45` (GREEN)

**3. [Rule 1 - Bug] Collapsed `MavMessage::LOG_REQUEST_END(_) => …` into catch-all**
- **Found during:** Task 2 (clippy pass)
- **Issue:** Initial `drive_once` implementation had both `MavMessage::LOG_REQUEST_END(_) => Vec::new()` and `_ => Vec::new()` arms. Clippy `match-same-arms` rejected this since the observable behaviour is identical.
- **Fix:** Collapsed `LOG_REQUEST_END` into the catch-all `_ => Vec::new()` arm with a documentation comment noting this is intentional per MAVLink common.xml §LOG_REQUEST_END (msgid 122, FC releases state without replying).
- **Files modified:** `crates/roz-mavlink/tests/common/mock_log_transport.rs`
- **Verification:** `cargo clippy -p roz-mavlink --tests -- -D warnings` clean; `drive_once_on_log_request_end_returns_empty` test still passes.
