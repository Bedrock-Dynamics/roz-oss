#[derive(Debug, Clone, serde::Deserialize, serde::Serialize)]
pub struct UlogConfig {
    #[serde(default = "default_enabled")]
    pub enabled: bool,
    #[serde(default = "default_download_timeout_secs")]
    pub download_timeout_secs: u64,
    #[serde(default)]

### `crates/roz-worker/src/config.rs` (MODIFIED)

Added `pub ulog: crate::ulog_config::UlogConfig` field immediately after `pub mavlink: MavlinkConfig`, with `#[serde(default)]` so absent `[ulog]` sections yield `UlogConfig::default()`.

Added two `figment::Jail` tests in the existing `#[cfg(test)] mod tests`:
- `config_loads_nested_ulog_overrides_from_env` — sets `ROZ_ULOG__ENABLED=false`, `ROZ_ULOG__DOWNLOAD_TIMEOUT_SECS=120`, `ROZ_ULOG__KEEP_FC_COPY=true` via `figment::Jail`, asserts all three override
- `config_ulog_defaults_when_section_missing` — only required env vars set, asserts the three documented defaults

**No `WorkerConfig::load()` body change was required** — the existing `Env::prefixed("ROZ_").split("__")` provider handles `ROZ_ULOG__*` nesting for free (precedent: Phase 26.7 Plan 04 added `observability.copper` identically).

## Key Decisions

**D-07 placement honored:** `ulog` is a top-level field on `WorkerConfig`, NOT nested under `observability`. Ulog auto-download is a distinct subsystem from MCAP/copper observability and deserves its own operator-facing config surface.

**`#[serde(default)]` everywhere:** Both the field (so absent TOML section works) and each struct field (so partial TOML sections work) carry `#[serde(default)]`. This matches the `ObservabilityCopperConfig` pattern exactly.

**Const-fn defaults, not literals:** Mirrors the analog — `default_enabled()` and `default_download_timeout_secs()` are `const fn` so they can be referenced from `#[serde(default = "...")]` attributes and from the `Default` impl without duplication.
