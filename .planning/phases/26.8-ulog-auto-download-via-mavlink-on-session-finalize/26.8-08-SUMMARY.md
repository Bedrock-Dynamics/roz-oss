## Decisions Made

- **Harness split**: Option A (`test-helpers` feature on roz-mavlink) + Option (b) (duplicate the mock into roz-worker). Rejected promoting MockLogTransport to a `pub mod testing` in roz-mavlink's library because it would move ~260 LOC from `tests/` to `src/` and publicly expose mavlink-specific types that belong in test-only code. The ~260 LOC duplication is easier to audit, and drift risk is caught by the pinned `FIXTURE_DIGEST_SHA256_HEX` in both copies.
- **`build_server(true)` flip**: Plan did not anticipate this. Worker's `build.rs` only had `build_client(true)` for Phase 26.7 (worker dials roz-server). The integration test needed `ArtifactServiceServer` to host a mock without depending on roz-server's database + auth scaffolding. The honest cost is slightly larger worker object files in release artifacts (generated server stubs are dead code at runtime) — a feature-gated split would complicate Cargo's crate-level dep graph across the test + release boundary. Tracked as Rule 3 deviation below.
- **Observer channel pattern**: the replay task reads outbound MAVLink frames, taps each into an observer mpsc, THEN drives the mock. This lets the test body assert on the exact LOG_* frame sequence AFTER `finalize_ulog_archive` completes (including the final `LOG_ERASE`). Without the observer tap, the replay task would consume frames before the test could read them.
- **100ms settle sleep between drop(backend) and replay_handle.abort()**: gives the replay task time to drain any in-flight `LOG_ERASE` frame before the task is aborted. Without the sleep, `send_log_erase` enqueues LOG_ERASE to the outbound channel but the replay task may be cancelled before it reads + forwards the frame to the observer.

## Issues Encountered

- **Type inference on `observed_tx.send(msg.clone())`**: initial implementation passed `msg.clone()` before `mock.drive_once(&msg)` borrowed it; rustc could not infer the clone's type. Fixed by reordering — drive the mock first, THEN forward the owned `msg` to the observer (no clone needed).
- **`drop(harness.backend)` + replay-task race**: the final `LOG_ERASE` frame in the happy-path test was occasionally not observed when the test body raced `replay_handle.abort()` against the replay task's frame-draining loop. Fixed with a 100ms settle sleep between the drop and the abort, giving the replay task time to move LOG_ERASE from the outbound receiver into the observer channel.
- **Clippy rejected `_log_tx` underscore prefix**: per `clippy::used-underscore-binding`, a field is either intentionally unused (prefix `_`) or actually referenced (no prefix). Renamed to drop the underscore; the Harness struct's doc comment now explicitly states "kept alive so the log broadcast channel stays open."
