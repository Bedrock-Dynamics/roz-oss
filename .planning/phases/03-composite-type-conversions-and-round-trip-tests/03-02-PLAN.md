Core proptest macros:
```rust
use proptest::prelude::*;
use proptest::collection;

prop_compose! {
    fn arb_type()(field1 in strategy1, field2 in strategy2) -> DomainType {
        DomainType { field1, field2 }
    }
}

proptest! {
    #[test]
    fn roundtrip_type(val in arb_type()) {
        let proto = roz_v1::ProtoType::from(&val);
        let back = DomainType::try_from(proto).unwrap();
        prop_assert_eq!(val, back);
    }
}
```

Key strategy choices for this domain:
- Floats: Use `-1e6f64..1e6` or similar finite ranges (NOT `any::<f64>()` which generates NaN/Inf)
- Strings: Use `"[a-z_]{3,20}"` for names/identifiers
- Enums: Use `0u8..N` mapped to variants
- Vecs: Use `collection::vec(arb_inner(), 0..3)` for bounded collections
- BTrees: Use `collection::btree_map(arb_key(), arb_val(), 0..3)`
- Option: Use `proptest::option::of(strategy)`
- DateTime: Use constrained range `946684800i64..4102444800` (2000-01-01 to 2100-01-01) then `DateTime::from_timestamp(secs, 0).unwrap()`
- Digest strings: Use `"[a-f0-9]{64}"` (SHA-256 hex format)

**CRITICAL constraints:**
- All f64 strategies MUST use finite ranges like `-1e6f64..1e6` — never `any::<f64>()` (generates NaN which breaks PartialEq)
- DateTime strategies MUST use `946684800i64..4102444800` (year 2000-2100) to avoid negative timestamp edge cases
- Digest strings use `"[a-f0-9]{64}"` — per D-02, they are opaque pass-through
- FrameTree strategy MUST produce valid trees (root first, then children referencing existing parents)
- SensorCalibration scale: test both None (no scale vec) and Some(vec) cases — the strategy should use `proptest::option::of(collection::vec(...))` for the scale field
- CalibrationOverlay temperature_range: use `proptest::option::of((min..max_pair))` — both min and max must be present or both absent
- EmbodimentRuntime joint_count/tcp_count: constrain to `0u32..1000` to avoid u32::MAX edge case in proptest generation (realistic robot values)

**Round-trip test pattern for each type:**
```rust
proptest! {
    #[test]
    fn roundtrip_TYPE(val in arb_TYPE()) {
        let proto = roz_v1::ProtoType::from(&val);
        let back = DomainType::try_from(proto).unwrap();
        prop_assert_eq!(val, back);
    }
}
```

For types where domain->proto uses `From` (infallible) and proto->domain uses `TryFrom`, the `.unwrap()` is correct — if TryFrom fails on a valid domain value, the conversion is buggy.

**Digest assertion pattern (for CONV-04 types):**
In addition to the main round-trip, add explicit digest field assertions where digest fields exist:
```rust
// Inside the proptest! block for embodiment_model:
prop_assert_eq!(val.model_digest, back.model_digest, "model_digest must round-trip as opaque string");
```
This is technically redundant with `prop_assert_eq!(val, back)` but makes the CONV-04 requirement explicitly tested and visible.
