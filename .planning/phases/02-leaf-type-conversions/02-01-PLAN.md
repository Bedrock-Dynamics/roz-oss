**Module doc comment:** `//! Bidirectional conversions between generated embodiment protobuf types and roz-core domain types.`

**Error type (per D-03, CONV-06):**
```rust
#[derive(Debug, thiserror::Error)]
pub enum EmbodimentConvertError {
    #[error("missing required field: {0}")]
    MissingField(String),
    #[error("invalid enum value for {type_name}: {value}")]
    InvalidEnum { type_name: &'static str, value: i32 },
    #[error("missing oneof variant: {0}")]
    MissingOneOf(String),
    #[error("invalid timestamp: out of range")]
    InvalidTimestamp,
}
```
Note: Omit `InvalidQuaternion` variant -- keep conversions purely structural per research assumption A1.

**Timestamp helpers:**
- `pub(crate) fn datetime_to_proto(ts: chrono::DateTime<chrono::Utc>) -> prost_types::Timestamp` -- seconds from `ts.timestamp()`, nanos from `ts.timestamp_subsec_nanos()` cast to i32
- `pub(crate) fn proto_to_datetime(ts: prost_types::Timestamp) -> Result<chrono::DateTime<chrono::Utc>, EmbodimentConvertError>` -- use `chrono::DateTime::from_timestamp(ts.seconds, u32::try_from(ts.nanos).unwrap_or(0))` returning `InvalidTimestamp` on `None`

**Geometry primitives (per D-02, CONV-02):**

1. `[f64; 3]` <-> `roz_v1::Vec3`: Both directions are `From` (infallible). `From<&[f64; 3]> for roz_v1::Vec3` maps `v[0]->x, v[1]->y, v[2]->z`. `From<roz_v1::Vec3> for [f64; 3]` maps `[v.x, v.y, v.z]`.

2. `[f64; 4]` (WXYZ) <-> `roz_v1::Quaternion` (XYZW): Both `From`. Domain->Proto: `x: rotation[1], y: rotation[2], z: rotation[3], w: rotation[0]`. Proto->Domain: `[q.w, q.x, q.y, q.z]`. Add explicit comments on every index mapping line.

3. `Transform3D` <-> `roz_v1::Transform3D`: `From<&Transform3D> for roz_v1::Transform3D` wraps translation/rotation in `Some(...)` sub-messages, maps `timestamp_ns` directly. `TryFrom<roz_v1::Transform3D> for Transform3D` extracts `.translation.ok_or_else(MissingField)` and `.rotation.ok_or_else(MissingField)`, applies XYZW->WXYZ swap on rotation.

4. `Inertial` <-> `roz_v1::Inertial`: `From<&Inertial> for roz_v1::Inertial` maps `mass` and `center_of_mass` via Vec3 From. `TryFrom<roz_v1::Inertial> for Inertial` extracts `center_of_mass.ok_or_else(MissingField)`.

**Simple enum conversions (per CONV-01, all 7 enums):**

For each of these 7 enums, implement a helper function pair rather than trait impls (avoids orphan rule issues with i32):
- `fn domain_joint_type_to_proto(jt: &JointType) -> i32`
- `fn proto_to_domain_joint_type(value: i32) -> Result<JointType, EmbodimentConvertError>`

The enums and their mappings:
1. `JointType`: Revolute/Prismatic/Fixed/Continuous <-> JOINT_TYPE_REVOLUTE/PRISMATIC/FIXED/CONTINUOUS
2. `TcpType`: Gripper/Tool/Sensor/Custom <-> TCP_TYPE_GRIPPER/TOOL/SENSOR/CUSTOM
3. `SensorType`: JointState/ForceTorque/Imu/Camera/PointCloud/Other <-> SENSOR_TYPE_JOINT_STATE/FORCE_TORQUE/IMU/CAMERA/POINT_CLOUD/OTHER
4. `ZoneType`: Allowed/Restricted/HumanPresence <-> ZONE_TYPE_ALLOWED/RESTRICTED/HUMAN_PRESENCE
5. `FrameSource`: Static/Dynamic/Computed <-> FRAME_SOURCE_STATIC/DYNAMIC/COMPUTED
6. `BindingType`: JointPosition/JointVelocity/ForceTorque/Command/GripperPosition/GripperForce/ImuOrientation/ImuAngularVelocity/ImuLinearAcceleration <-> BINDING_TYPE_*

**Oneof conversions (per CONV-01):**

1. `Geometry` <-> `roz_v1::Geometry`: `From<&Geometry> for roz_v1::Geometry` creates the appropriate `roz_v1::geometry::Shape` variant. Box maps `half_extents[0..3]` to `half_extent_x/y/z`. Mesh maps `scale: Option<[f64;3]>` to `optional Vec3`. `TryFrom<roz_v1::Geometry> for Geometry` extracts `.shape.ok_or_else(MissingOneOf("Geometry.shape"))`, matches each variant.

2. `WorkspaceShape` <-> `roz_v1::WorkspaceShape`: Similar to Geometry. Uses `WorkspaceBoxShape/WorkspaceSphereShape/WorkspaceCylinderShape`. Note Cylinder uses `half_height` (not `length`).

3. `SemanticRole` <-> `roz_v1::SemanticRole`: `From<&SemanticRole> for roz_v1::SemanticRole` maps each variant to the oneof role. `PrimaryManipulatorJoint{index}` -> `Role::PrimaryManipulatorJoint(ManipulatorJointRole{index})`. Empty variants (PrimaryGripper, etc.) -> `Role::PrimaryGripper(EmptyRole{})`. Custom{role} -> `Role::Custom(CustomRole{role})`. `TryFrom` reverses, extracting `.role.ok_or_else(MissingOneOf("SemanticRole.role"))`.
