```rust
use sqlx::PgPool;
use uuid::Uuid;

/// Row type for embodiment data queries. Contains host identity + JSONB columns.
#[derive(Debug, sqlx::FromRow)]
pub struct EmbodimentRow {
    pub id: Uuid,
    pub tenant_id: Uuid,
    pub embodiment_model: Option<serde_json::Value>,
    pub embodiment_runtime: Option<serde_json::Value>,
}

/// Fetch embodiment data for a host by its primary key.
pub async fn get_by_host_id(pool: &PgPool, host_id: Uuid) -> Result<Option<EmbodimentRow>, sqlx::Error> {
    sqlx::query_as::<_, EmbodimentRow>(
        "SELECT id, tenant_id, embodiment_model, embodiment_runtime FROM roz_hosts WHERE id = $1"
    )
    .bind(host_id)
    .fetch_optional(pool)
    .await
}

/// Upsert embodiment data for a host. Sets model and optionally runtime.
/// Only updates if the host exists.
pub async fn upsert(
    pool: &PgPool,
    host_id: Uuid,
    model: &serde_json::Value,
    runtime: Option<&serde_json::Value>,
) -> Result<bool, sqlx::Error> {
    let result = sqlx::query(
        "UPDATE roz_hosts SET embodiment_model = $2, embodiment_runtime = COALESCE($3, embodiment_runtime), updated_at = now() WHERE id = $1"
    )
    .bind(host_id)
    .bind(model)
    .bind(runtime)
    .execute(pool)
    .await?;
    Ok(result.rows_affected() > 0)
}
```

**Trait impl -- all 4 RPCs:**

`get_model`: Authenticate -> parse host_id UUID -> `roz_db::embodiments::get_by_host_id()` -> verify `row.tenant_id == tenant_id` (return NOT_FOUND if mismatch for tenant isolation) -> extract `embodiment_model` JSONB (return `FAILED_PRECONDITION` with "host has no embodiment model" if None) -> `serde_json::from_value::<roz_core::embodiment::model::EmbodimentModel>()` (return INTERNAL on failure) -> `.into()` to convert domain to proto -> `Response::new(proto_model)`.

`get_runtime`: Same pattern as get_model but extract `embodiment_runtime` JSONB -> deserialize as `roz_core::embodiment::embodiment_runtime::EmbodimentRuntime` -> `.into()`.

`list_bindings`: Authenticate -> parse host_id -> query DB -> verify tenant -> extract `embodiment_model` JSONB -> deserialize as `EmbodimentModel` -> convert `model.channel_bindings` to proto via `.into_iter().map(Into::into).collect()` -> `ListBindingsResponse { bindings }`.

`validate_bindings`: Authenticate -> parse host_id -> query DB -> verify tenant -> extract `embodiment_runtime` JSONB (FAILED_PRECONDITION if None) -> deserialize as `EmbodimentRuntime` -> extract validation inputs from domain types:
  - `joint_names: Vec<&str>` from `runtime.model.joints.iter().map(|j| j.name.as_str())`
  - `sensor_ids: Vec<&str>` from `runtime.model.sensor_mounts.iter().map(|s| s.sensor_id.as_str())`
  - `frame_ids: Vec<&str>` from `runtime.model.frame_tree.frames.keys().map(String::as_str)`
  - `channel_count` from `runtime.model.channel_bindings.len() as u32`
  - Call `roz_core::embodiment::binding::validate_bindings()` with these inputs
  - Convert result `Vec<UnboundChannel>` to proto: `.into_iter().map(Into::into).collect()`
  - Return `ValidateBindingsResponse { valid: proto_unbound.is_empty(), unbound_channels: proto_unbound }`

**Error mapping (gRPC status codes):**
- Missing/invalid auth -> `UNAUTHENTICATED`
- Invalid UUID -> `INVALID_ARGUMENT` with "host_id is not a valid UUID"
- Host not found or wrong tenant -> `NOT_FOUND` with "host not found"
- Host exists but no embodiment data -> `FAILED_PRECONDITION` with "host has no embodiment model" or "host has no embodiment runtime"
- JSONB deserialization failure -> `INTERNAL` with `format!("corrupt model data: {e}")`
- Database error -> `INTERNAL` with "database error" (log with tracing::error! first)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| gRPC client -> EmbodimentService | Untrusted client sends host_id and API key |
| EmbodimentService -> Database | Service queries tenant-scoped JSONB data |
| JSONB data -> domain types | Stored JSON deserialized into typed structs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | get_model/get_runtime | mitigate | Check `row.tenant_id == tenant_id` after every DB query; return NOT_FOUND (not FORBIDDEN) to avoid leaking host existence across tenants |
| T-04-02 | Spoofing | authenticated_tenant_id | mitigate | Require valid API key via GrpcAuth trait; reject with UNAUTHENTICATED on missing/invalid auth header |
| T-04-03 | Tampering | JSONB deserialization | mitigate | Deserialize through typed `serde_json::from_value::<EmbodimentModel>()` which rejects arbitrary/malformed JSON; return INTERNAL on failure |
| T-04-04 | Elevation of Privilege | validate_bindings | mitigate | Same tenant isolation check on validate_bindings -- cannot validate another tenant's host bindings |
| T-04-05 | Denial of Service | get_model/get_runtime | accept | Large JSONB blobs (10-100KB) are within normal Postgres TOAST handling; no special rate limiting beyond existing server infrastructure |
</threat_model>
