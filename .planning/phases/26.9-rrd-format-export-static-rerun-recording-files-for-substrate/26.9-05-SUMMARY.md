# Phase 26.9 Plan 05: emit_tf + emit_pose Transform3D mapping Summary

Both functions decode Foxglove proto messages via `prost::Message::decode`, narrow `f64 → f32` translation/rotation components, and log a `rerun::archetypes::Transform3D` archetype via `rec.log(...)`. ## Quaternion Convention (RESEARCH §Topic 5)

Foxglove `Quaternion` proto fields are tagged `1=x, 2=y, 3=z, 4=w` (verified against `proto/foxglove/Quaternion.proto:11-19`). Rerun's `rerun::Quaternion::from_xyzw([x, y, z, w])` accepts the same `(x, y, z, w)` order. Plan 05 performs a **direct field copy** with no reordering. A wrong-order swap would silently produce near-identity rotations for small angles — exactly the kind of bug a careless developer might miss in unit tests but a robotics operator would notice as drift in 3D playback.
