## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Added `dead_code` to foxglove `#[allow]` set**

- **Found during:** Task 3 verification (`cargo build -p roz-cli --features export-rrd`)
- **Issue:** Plan 02 mounts the entire foxglove proto module but the skeleton has no consumers yet (Plans 05/06 will use them). Without `dead_code`, rustc emits 5 "struct never constructed" warnings, which fails the acceptance criterion "no warnings from roz-cli source" and would also fail CI's `cargo clippy --workspace -- -D warnings`.
- **Fix:** Added `dead_code` to the existing `#[allow(...)]` block on `pub(crate) mod foxglove`, with reason text noting Plans 05/06 will consume the types and the warnings will self-clear once consumers exist.
- **Files modified:** `crates/roz-cli/src/commands/mcap/mod.rs`
- **Commit:** `1e79d4e`

**2. [Rule 1 - Bug] Added `#[expect(clippy::unused_async)]` on placeholder `to_rrd` functions**

- **Found during:** Task 3 verification (`cargo clippy -p roz-cli` and `cargo clippy -p roz-cli --features export-rrd`)
- **Issue:** Both `#[cfg(not(feature = "export-rrd"))] async fn to_rrd` and `#[cfg(feature = "export-rrd")] async fn to_rrd` are Plan 02 placeholders that immediately `bail!` — clippy::pedantic warns `unused_async`. Workspace policy is `clippy::pedantic = warn` and CI runs `-D warnings`, so this would gate the build.
- **Fix:** Added targeted `#[expect(clippy::unused_async, reason = "Plan 02 skeleton; Plans 03/04/05/06/07 add async RecordingStream + tokio file I/O")]` per CLAUDE.md convention. Plans 03-07 will replace the bodies with real async I/O and remove the `#[expect]` (or `#[expect]` will fail as `unfulfilled_lint_expectation` and force the cleanup).
- **Files modified:** `crates/roz-cli/src/commands/mcap/mod.rs`
- **Commit:** `1e79d4e`

**3. [Rule 3 - Blocking] Reformatted overlong arg attributes via `cargo fmt`**

- **Found during:** Final `cargo fmt --check -p roz-cli`
- **Issue:** A few `#[arg(...)]` attributes and the `parse_mcap_to_rrd_rejects_missing_output_in_single_mode` assertion were split across multiple lines but rustfmt (with the project's 120-col `max_width`) preferred them collapsed onto single lines. The pre-commit hook runs `cargo fmt --check`, so this would block.
- **Fix:** Ran `cargo fmt -p roz-cli` to apply the project rustfmt config; verified `cargo fmt --check -p roz-cli` is clean.
- **Files modified:** `crates/roz-cli/src/commands/mcap/mod.rs`
- **Commit:** `1e79d4e` (folded into the Task 3 commit alongside the wiring + lint fixes)

## Known Stubs

The five submodule files (`export.rs`, `recording.rs`, `transforms.rs`, `text_logs.rs`, `camera.rs`) are intentional stubs — they exist solely so Plans 03-07 can land their implementations in parallel without file-conflict races. Each file:

- Is gated by `#![cfg(feature = "export-rrd")]` so the default build never sees its (currently empty) contents.
- Has a module-level doc comment naming the downstream plan that owns the file.
- Compiles to nothing (the cfg attribute makes the empty body a no-op).

The `to_rrd` dispatcher in `mod.rs` returns `anyhow::bail!("Phase 26.9 Plan 02 skeleton: ...")` under `#[cfg(feature = "export-rrd")]` until Plan 03 (or whichever plan first lands) replaces the body. This is the intentional Plan 02 deliverable per the plan's `<NOTE>` block on Task 2 ("`cargo build --features export-rrd` at the END of Plan 02 will FAIL ... — that is expected and acceptable" was the original plan text; we resolved this with the `bail!` placeholder so both feature variants now compile clean rather than only the default).

## Followups (Flag, Not Block)

- The `#[expect(clippy::unused_async)]` annotations on both `to_rrd` functions will start emitting `unfulfilled_lint_expectation` warnings once Plans 03/04 wire actual `.await` calls. That is the intended behavior — it forces those plans to remove the annotations rather than letting them stagnate.
- The `dead_code` allow on the foxglove module similarly should be removed (or narrowed) once Plans 05/06 actually consume `FrameTransform`, `PoseInFrame`, and `Log`. Recommend Plan 05 (transforms) is the natural place to revisit this.
