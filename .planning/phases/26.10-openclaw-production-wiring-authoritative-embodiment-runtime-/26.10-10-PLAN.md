## Approach

**Architectural decision: extend the existing entry point (Codex-style trade-off note).**

There are two ways to wire `latch_persist_tx` from worker boot to the controller loop:

1. Add a parallel entry point `spawn_with_policy_and_io_and_latch_persist` on `CopperHandle`.
2. Extend `spawn_with_policy_and_io`'s signature with an optional `latch_persist_tx` param.

**Choice: option 2.** There is exactly one production caller (`crates/roz-worker/src/main.rs:754`); all internal callers of `run_controller_loop_with_policy` (verified today: `controller.rs:2097` fallback wrapper plus 12 in-file test sites at :3740, :3788, :3871, :4086, :4154, :4224, :4305, :4432, :4532, :4613, :4981, :5089, plus `handle.rs:193` / `:209` controller-thread invocations) already pass through wrapper functions or test bodies that can default to `None`. `run_controller_loop_with_policy` already has `#[allow(clippy::too_many_arguments)]`. Adding a parallel entry doubles the test surface and creates two paths to maintain. Extending the signature keeps the path count at 1, and the compiler enumerates every site that needs updating after the signature change (no manual grep-counting).

**Subscriber routing decision: worker-level `shared_cmd_tx` slot.**

The `CopperHandle` is per-task (created at main.rs:754, dropped on task end), but NATS subscribers must live for the worker's lifetime — re-subscribing per task is wasteful and breaks the policy-push / resume-instruction precedent at main.rs:1945-:2155. Mirror the `shared_copper_state: Arc<ArcSwap<Option<...>>>` pattern at main.rs:518 / 1586 / 771: add `shared_cmd_tx: Arc<ArcSwap<Option<mpsc::Sender<ControllerCommand>>>>`, install on task spawn, clear on task end, and have subscribers `load()` it on every message. If `None` (no live controller), log + drop the message (benign — an operator-sent ack to a worker with no active task is not an error).

**Boot ordering invariant.**

At `main.rs:754` (the per-task OodaReAct branch, inside `async fn execute_task` declared at :479; the in-scope WAL parameter is `worker_wal_shared` from :506):

1. `let initial_latch = worker_wal_shared.as_ref().map(|w| w.load_latch_state()).transpose()?;` — corruption Err propagates as task-level failure (FW-05 contract: NO silent Run fallback).
2. `let (latch_tx, latch_rx) = std::sync::mpsc::sync_channel::<LatchState>(16);` — bounded; matches the SyncSender type already wired in controller.rs:2172.
3. `let handle = CopperHandle::spawn_with_policy_and_io(..., Some(latch_tx));` — spawns Copper with persistence wired.
4. **Immediately** seed the latch into ArcSwap: `handle.state().rcu(|s| { let mut new = (**s).clone(); new.latch_state = initial_latch.unwrap_or(LatchState::Run); Arc::new(new) });` — the controller thread's first iteration reads via `shared_state.load()`. Race is benign (no controllers promoted yet) but the ordering matters.
5. `shared_cmd_tx.store(Arc::new(Some(handle.cmd_tx())));` — installs for the worker-level subscribers.
6. `tokio::task::spawn_blocking({ let wal = worker_wal_shared.clone(); move || while let Ok(s) = latch_rx.recv() { let _ = wal.save_latch_state(s).map_err(|e| tracing::error!(error = %e, "WAL save_latch_state failed")); } });` — `spawn_blocking` because `WalStore` is `rusqlite` (sync).
7. On task end (drop / shutdown): `shared_cmd_tx.store(Arc::new(None));` — drainer task ends naturally when `latch_tx` is dropped (sender count reaches zero, recv returns Err).
