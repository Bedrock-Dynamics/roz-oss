## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Information Disclosure | NATS subject with host_id | mitigate | Streaming RPC handlers verify tenant ownership via `fetch_embodiment_row` before subscribing (Plan 02). NATS subject is internal-only, not exposed to external clients. |
| T-07-02 | Denial of Service | NATS publish in REST handler | accept | Publish is fire-and-forget with warn on failure. Failed publish does not block REST response. No amplification risk. |
| T-07-03 | Tampering | NATS event payload | mitigate | Streaming handlers re-read from DB on every NATS notification (Plan 02). NATS event is a notification trigger only, not a data carrier for responses. |
| T-07-04 | Spoofing | Internal NATS subject | accept | NATS subjects use `roz.internal.*` prefix. Internal network only. NATS auth (operator seed) controls access. |
