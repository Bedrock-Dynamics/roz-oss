# Phase 26.5 Plan 01: Vendor Foxglove Multimedia Protos Summary

> **Note:** Reconstructed retroactively from git history on 2026-04-24 during a /gsd-health backfill pass. The original execution did not write a summary; phase 26.5 verification (7/7 SC pass) covers the full delivery.

Vendor 24 Foxglove multimedia .proto files into `proto/foxglove/`, extend `roz-server/build.rs` to compile the 6 target schemas, register `SCHEMA_*` + Phase 29+ `CHANNEL_*` constants in `observability/mod.rs`, and expose prost Message types via a new `foxglove_types` submodule.

## What was built

### Vendored protos (commit a6ddbc4, 847 insertions, 25 files)

**6 target schemas:**
- `CompressedVideo.proto` — H.264 target per R-01
- `CompressedImage.proto`
- `RawImage.proto`
- `PointCloud.proto`
- `SceneUpdate.proto`
- `ImageAnnotations.proto`

**18 transitive deps:**
ArrowPrimitive, CircleAnnotation, Color, CubePrimitive, CylinderPrimitive, KeyValuePair, LinePrimitive, ModelPrimitive, PackedElementField, Point2, Point3, PointsAnnotation, SceneEntity, SceneEntityDeletion, SpherePrimitive, TextAnnotation, TextPrimitive, TriangleListPrimitive.

Upstream pin: `foxglove-sdk @ 7c7a179e32eff0855f94b83d177affcc1709ee32` — provenance recorded in `proto/foxglove/README.md` along with a re-vendor procedure.

### build.rs extension (commit 0f28cc1)

Extends the second `tonic_build::configure()` call with the 6 new schema names. Transitive deps auto-resolve via the `proto/` include path. `build_server(false) + build_client(false)` retained — `prost::Message` types are still emitted per tonic-build contract (research A6).

### Schema + channel registration (commit ba9bf8b)

`crates/roz-server/src/observability/mod.rs` gains:

- 6 new `SCHEMA_*` constants: `SCHEMA_COMPRESSED_VIDEO`, `SCHEMA_COMPRESSED_IMAGE`, `SCHEMA_RAW_IMAGE`, `SCHEMA_POINT_CLOUD`, `SCHEMA_SCENE_UPDATE`, `SCHEMA_IMAGE_ANNOTATIONS`.
- 3 future-producer `CHANNEL_*` constants for Phase 29+ perception: `CHANNEL_POINTCLOUD`, `CHANNEL_SCENE_UPDATE`, `CHANNEL_ANNOTATIONS`.
- New `foxglove_types` submodule (`crates/roz-server/src/observability/foxglove_types.rs`) exposing prost `Message` types via `tonic::include_proto!("foxglove")` for downstream encode/decode.

### Codegen smoke tests (commit 6f07755)

Verifies `build_server=false, build_client=false` still emits `prost::Message` types for all 6 multimedia schemas — guards against future tonic-build behaviour drift.
