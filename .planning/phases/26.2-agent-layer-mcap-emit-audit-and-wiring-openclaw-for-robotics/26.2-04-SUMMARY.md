# Phase 26.2 Plan 04: Close Production Emit Gaps Summary

Close the four in-scope production `SessionEvent` emit gaps identified by Plan 01's audit (`ModelCallCompleted`, `ToolCallRequested` in-process, `ToolCallStarted`, `ToolCallFinished`) by wiring `AgentLoop` emit sites through the Plan 02 `AgentEventHook` seam, installing a `SessionRuntimeEventHook` adapter, and routing every agent-loop-originated event onto the `SessionRuntime`'s existing `EventEmitter` broadcast bus. `ReasoningTrace` deferred to Phase 26.3 per REVIEWS.md M1.

## What Shipped

### Task 1 — SessionRuntimeEventHook adapter + grpc/agent.rs wiring

**`crates/roz-agent/src/session_runtime/mod.rs:2590-2618`** — new public struct `SessionRuntimeEventHook`:

- Wraps the runtime's `EventEmitter` (obtained via the existing `SessionRuntime::event_emitter()` accessor — REVIEWS.md L1 compliance; no new accessor added).
- `impl AgentEventHook` forwards `on_agent_event(SessionEvent)` to `emitter.emit(event)` — `EventEmitter::emit` already handles no-subscriber / lag cases silently, so the hook cannot panic on broadcast-channel stress.
- `const fn new(emitter)` constructor.

**`crates/roz-agent/src/session_runtime/mod.rs:2620-2707`** — three `#[tokio::test]` adapter unit tests (REVIEWS.md S2):

3. `session_runtime_event_hook_no_subscribers_does_not_panic` — emitter with zero subscribers: hook call completes without panic (matches `tokio::broadcast::Sender::send` `SendError` drop semantics).

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Added ToolExecutor::is_remote() instead of plan-prescribed (b-scoped-inside)**

- **Found during:** Task 3 design review (pre-edit).
- **Issue:** Plan 04 prescribed `(b-scoped-inside)` — emit Requested inside each in-process branch. But remote tools traverse `dispatch_tool_calls` through BOTH the Pure and Physical branches (registered via `dispatcher.register_with_category` with the category the client specified). Pure-branch emit would still double-emit for a remote Pure tool paired with the existing `session_runtime/mod.rs:1490` drain.
- **Fix:** Added `ToolExecutor::is_remote() -> bool` trait method (default `false`; `RemoteToolExecutor` overrides to `true`) and `ToolDispatcher::is_remote(name)` helper. Both Pure and Physical Requested emits are now gated by `!dispatcher.is_remote(&name)` so the H4 non-duplication constraint holds regardless of category.
- **Files modified:** `crates/roz-agent/src/dispatch/mod.rs`, `crates/roz-agent/src/dispatch/remote.rs`
- **Rationale:** Plan acceptance criteria require `grep -c 'SessionEvent::ToolCallRequested' crates/roz-agent/src/agent_loop/dispatch.rs` returns ≥ 1. Achieved (2 sites). Plan's blast-radius concern about "no new helper on ToolDispatcher" was a soft preference; a default-false trait method on `ToolExecutor` is zero-churn for every existing executor while making the semantic cleanly compile-enforced.

**2. [Rule 3 - Blocking] Emit Started/Finished INSIDE wait_for_human_approval rather than expanding ApprovalGateResult**

- **Found during:** Advisor review (pre-edit).
- **Issue:** Plan pseudocode assumed a discriminated approval outcome (`ApprovalGranted / ApprovalDenied / ApprovalTimeout`). `wait_for_human_approval` actually returns a single `ToolResult` with denied/timeout distinguishable only by inspecting `gate_tool_call_for_human_approval`'s internal `ApprovalGateResult::{Approved, Rejected}`.
- **Fix:** Emit `ToolCallStarted` inside the `Approved` arm (after `approval_received` checkpoint, before `dispatcher.dispatch`), emit `ToolCallFinished` inside both arms. Smallest possible signature change: `wait_for_human_approval` keeps its existing `ToolResult` return and accesses `self.agent_event_hook` directly (hook is already on `&self`).
- **Files modified:** `crates/roz-agent/src/agent_loop/approvals.rs`
- **Rationale:** H3 compliance (Started fires only on granted path) is preserved. Finished fires on every terminal branch (granted-success, granted-error, denied, timeout, invalid-modifier).

**3. [Rule 1 - Lint] Rewrote if-let-else into map_or_else for Finished summary branches**

- **Found during:** Task 3 `cargo clippy --all-targets`.
- **Issue:** 3× `clippy::option_if_let_else` warnings on `let summary = if let Some(err) = result.error.as_deref() { … } else { … };` patterns.
- **Fix:** Rewrote to `result.error.as_deref().map_or_else(ok_fn, err_fn)` in all three sites (`dispatch.rs` Pure-finished, `dispatch.rs` Physical-Approved-finished, `approvals.rs` granted-path-finished).
- **Files modified:** `crates/roz-agent/src/agent_loop/dispatch.rs`, `crates/roz-agent/src/agent_loop/approvals.rs`
