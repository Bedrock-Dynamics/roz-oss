## Reviews Addressed

This plan is a REVIEWS-driven revision. Codex (gpt-5.5) findings driving the revision:

- **H3 (HIGH) ŌĆö Latched e-stop is not fail-closed at loop level.** Original plan put `latch_state` on `PreparedController` struct (per-controller). Codex review finding: the actual Copper loop has active/candidate/rollback controllers; existing rollback clears `estop_reason` (controller.rs:261); `ControllerState` has no latch field; `Resume` already exists with broader semantics. Per-controller latch is not fail-closed. **FIX:** Move latch to `ControllerState` (system-level), introduce explicit `AckEstop` + `ResumeAfterZeroVerified` commands distinct from existing `Resume`, make WAL the boot authority, define remain-latched-without-sensor behavior, and split per-channel-kind zero policy.
- **H3 ŌĆö Position-channel zero collision.** Universal `0.0` is safe for velocity/torque but commands "go to zero" for position channels (collision). **FIX:** Per-channel-kind zero policy table; position channels emit last commanded value (hold), velocity/torque/force emit `0.0`.
- **H3 ŌĆö Rollback path clears `estop_reason`.** Existing `controller.rs` rollback at line 261 unconditionally resets `estop_reason`. **FIX:** Guard rollback path with `if latch_state == Run` ŌĆö rollback only runs after latch cleared.
- **G1 ŌĆö `LatchState::Default = Run`** is required so `ControllerState::default()` (already derived) keeps working without a manual override.

## State Machine Diagram

```
                     [ Run ]
                       |
                  force_estop?
                       |
                       v
                   [ Latched ]  <----- (sticky on any e-stop assertion;
                       |                  emit per-channel-kind zero each tick)
              signed AckEstop arrives
                       |
                       v
                [ AwaitingAck ]  <---- (still emit zero; counting consecutive
                       |                  zero-motion ticks observed by sensor)
            N=10 consecutive zero-motion ticks
            AND sensor frame present each tick
                       |
                       v
                [ ZeroVerified ] <---- (still emit zero; ready for human resume)
                       |
            signed ResumeAfterZeroVerified arrives
                       |
                       v
                     [ Run ]

  FORBIDDEN transitions:
    Latched      -X-> Run            (no auto-rearm ŌĆö IEC 60204-1)
    Latched      -X-> ZeroVerified   (must pass through AwaitingAck)
    AwaitingAck  -X-> Run            (must verify zero first)
    Latched      -X-> Run via existing Resume (Resume's semantics unchanged;
                                                 it does not clear the latch)

  Sensor-absent behavior:
    [Latched | AwaitingAck] + sensor.try_recv() == None  -> stay in current state
                                                            (no motion verification possible)

  Boot behavior (WAL-authoritative):
    On worker restart, load_latch_state() reads WAL:
      LatchState::Run  ->  start in Run (normal boot)
      Otherwise        ->  start in the persisted state; emit zero each tick;
                            require AckEstop -> ZeroVerified -> ResumeAfterZeroVerified
                            to return to Run.
```

## Per-Channel-Kind Zero Policy (Safety-Critical Table)

When `latch_state.requires_zero_emission() == true`, each output channel emits according to its `BindingType` / `CommandInterfaceType` (read `crates/roz-core/src/embodiment/binding.rs:7-14, 49-54` for the canonical enums):

| Channel kind / BindingType | Latched-tick emission | Rationale |
|----------------------------|------------------------|-----------|
| `JointVelocity` | `0.0` | Zero velocity halts motion safely |
| `JointTorque` | `0.0` | Zero torque removes drive |
| `GripperForce` | `0.0` | Zero force releases grip |
| `ForceTorque` (sensor ŌĆö output side) | N/A ŌĆö sensor only | Read-only; not a command channel |
| `ImuAngularVelocity` (sensor) | N/A ŌĆö sensor only | Read-only |
| `JointPosition` | **Last commanded value** (hold) | `0.0` would command "move to position 0" ŌåÆ collision path |
| `GripperPosition` | **Last commanded value** (hold) | Same as JointPosition ŌĆö `0.0` could cause grip release / collision |
| `Command` (generic ŌĆö `BindingType::Command`) | `0.0` (default) ŌĆö but the policy is conservative; if `interface_type` carries a position semantic, fall through to position policy | Safety-default: emit `0.0` only when interface guarantees velocity/torque semantics |

**Implementation contract:** the latched-tick code MUST consult `ControlInterfaceManifest.bindings[i].binding_type` (or equivalent path through `EmbodimentRuntime` channel authority) to select the right zero-policy per channel. A test fixture asserts both kinds with one channel of each.

       //! FW-05 ŌĆö Latched e-stop state machine (IEC 60204-1 + EN ISO 13849-1).
       //!
       //! State machine (see plan diagram for full graph):
       //!   Run         -> Latched      : force_estop asserted (any source)
       //!   Latched     -> AwaitingAck  : signed safety.estop_ack.{worker_id} arrives
       //!   AwaitingAck -> ZeroVerified : N=10 consecutive zero-motion ticks AND sensor frame present
       //!   ZeroVerified -> Run         : signed safety.resume.{worker_id} arrives
       //!   Latched     -X-> Run         : FORBIDDEN per IEC 60204-1 (no auto-rearm)
       //!
       //! WAL-authoritative boot: on restart, load_latch_state() drives initial state.
       //! Sensor-absent: stay in current latched state until motion can be verified.

       use serde::{Deserialize, Serialize};

       /// Number of consecutive zero-motion ticks required to advance from
       /// AwaitingAck to ZeroVerified. At 100 Hz this is ~100 ms.
       pub const ZERO_VERIFY_TICK_COUNT: u32 = 10;

       /// Latched e-stop state. Auto-rearm is forbidden per IEC 60204-1.
       /// Default = Run so ControllerState::default() keeps working unchanged.
       #[derive(Debug, Clone, Copy, PartialEq, Eq, Default, Serialize, Deserialize)]
       #[serde(rename_all = "snake_case")]
       pub enum LatchState {
           #[default]
           Run,
           Latched,
           AwaitingAck,
           ZeroVerified,
       }

       impl LatchState {
           /// Whether this state should emit explicit zero CommandFrames each tick.
           #[must_use]
           pub const fn requires_zero_emission(self) -> bool {
               !matches!(self, Self::Run)
           }

           /// Apply an e-stop assertion. Always transitions to Latched (sticky).
           #[must_use]
           pub const fn assert_estop(self) -> Self {
               Self::Latched
           }

           /// Apply a signed AckEstop (distinct from existing Resume).
           /// Only valid from Latched.
           #[must_use]
           pub const fn apply_ack_estop(self) -> Self {
               match self {
                   Self::Latched => Self::AwaitingAck,
                   other => other,
               }
           }

           /// Advance after N consecutive zero ticks AND sensor frames present.
           /// Only valid from AwaitingAck.
           #[must_use]
           pub const fn apply_zero_verified(self) -> Self {
               match self {
                   Self::AwaitingAck => Self::ZeroVerified,
                   other => other,
               }
           }

           /// Apply a signed ResumeAfterZeroVerified.
           /// Only valid from ZeroVerified. Distinct from the existing
           /// ControllerCommand::Resume ŌĆö that variant retains its old semantics.
           /// Latched -> Run via this fn is a no-op (returns Latched).
           #[must_use]
           pub const fn apply_resume_after_zero_verified(self) -> Self {
               match self {
                   Self::ZeroVerified => Self::Run,
                   other => other,
               }
           }
       }

       mod tests {

           }

                   assert_eq!(state, back, "roundtrip failed for {state:?}");
               }
           }

           #[test]
           fn latch_state_requires_zero_emission() {
               assert!(!LatchState::Run.requires_zero_emission());
               assert!(LatchState::Latched.requires_zero_emission());
               assert!(LatchState::AwaitingAck.requires_zero_emission());
               assert!(LatchState::ZeroVerified.requires_zero_emission());
           }
