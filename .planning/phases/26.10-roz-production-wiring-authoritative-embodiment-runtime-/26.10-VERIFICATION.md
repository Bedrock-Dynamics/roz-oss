The hardcoded `let latch_persist_tx: Option<...> = None;` previously at :2172 is removed; 13 in-controller callers + 2 handle.rs callers + cross-crate test sites updated to pass `None`; the single production caller in `roz-worker/src/main.rs:806` passes `Some(latch_tx)`.
- **CR-01b (drainer + save_latch_state):** Worker boot at `main.rs:862-874` spawns a `tokio::task::spawn_blocking` drainer reading from `std::sync::mpsc::Receiver<LatchState>`; each received state is persisted via `worker_wal_shared.save_latch_state(state)`. Drainer ends naturally when the SyncSender drops on handle shutdown.
- **CR-01c (load_latch_state at boot + ArcSwap seed):** Worker boot at `main.rs:776-787` calls `wal.load_latch_state()` BEFORE `spawn_with_policy_and_io`. Corruption returns `Err` and propagates to `copper_spawn_failure` (NO silent Run fallback per IEC 60204-1). After the handle spawns and BEFORE the first controller tick, `handle.state().rcu(...)` writes the loaded `latch_state` into the `ControllerState` ArcSwap.
- **CR-02a + CR-02b (signed-NATS subscribers):** Worker boot at `main.rs:2342-2357` calls `roz_worker::safety_subscribers::spawn_estop_ack_subscriber` and `spawn_safety_resume_subscriber`. Each subscriber gates inbound payloads through Phase 23 `WorkerSigningContext::verify_inbound_worker`; verification failure publishes to `Subjects::safety_signature_failure_worker(host_id)` and DROPS the message.
- **CR-02c (verified messages dispatch via shared_cmd_tx):** The worker-level `shared_cmd_tx: Arc<ArcSwap<Option<mpsc::Sender<ControllerCommand>>>>` is constructed at boot (`main.rs:1722`); each OodaReAct task installs `Some(handle.cmd_tx())` after `spawn_with_policy_and_io` (`main.rs:853`) and clears to `None` on task drop / e-stop drop / shutdown (`main.rs:1473, 1535`). Subscribers `load()` the slot — `Some(tx)` forwards `AckEstop`/`ResumeAfterZeroVerified`; `None` logs and drops.
