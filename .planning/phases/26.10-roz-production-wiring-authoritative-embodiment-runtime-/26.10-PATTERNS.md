pub fn factory_for(family: Option<&str>) -> Option<Box<dyn IoFactory>> {
    match family {
        Some("manipulator") | Some("manipulator") => {
            Some(Box::new(crate::io_factory::FakeManipulatorFactory::default()))

### `crates/roz-copper/src/latch.rs` (NEW — `LatchState` enum)

**Analog:** `crates/roz-copper/src/channels.rs:31-89` (enum-with-serde shape).

**Pattern (planner-bound, grounded in IEC 60204-1 + EN ISO 13849-1 — see RESEARCH.md "Pattern 4"):**
```rust
use serde::{Deserialize, Serialize};

/// Latched e-stop state. Transitions are explicit and one-way unless
/// noted; auto-rearm is forbidden per IEC 60204-1.
#[derive(Debug, Clone, Copy, PartialEq, Eq, Default, Serialize, Deserialize)]
#[serde(rename_all = "snake_case")]
pub enum LatchState {
    #[default]
    Run,
    /// E-stop asserted; controller emits explicit zero frames every tick.
    Latched,
    /// Operator ack received via signed `safety.estop_ack.{worker_id}`.
    AwaitingAck,
    /// Sensor-confirmed zero motion observed for N consecutive ticks
    /// (N=10 @ 100 Hz = 100 ms). Awaits explicit `Resume` command.
    ZeroVerified,
}

impl LatchState {
    /// Whether this tick should emit explicit zero (vs. allow normal
    /// controller output).
    pub const fn requires_zero_emission(self) -> bool {
        !matches!(self, Self::Run)
    }
}
```

Transition logic (planner expands per RESEARCH.md "Pattern 4" diagram):
- `Run -> Latched` on any e-stop assertion (force_estop, safety_filter, watchdog, stale-heartbeat).
- `Latched -> AwaitingAck` on signed NATS `safety.estop_ack.{worker_id}` (Pitfall 4 — must verify Phase 23 sig).
- `AwaitingAck -> ZeroVerified` on N consecutive zero-motion ticks observed.
- `ZeroVerified -> Run` on signed `Resume` command.
- Persistence: per RESEARCH.md "Open Question 5" recommendation — persist `LatchState` in `crates/roz-worker/src/wal.rs` to survive worker restart fail-safe.
