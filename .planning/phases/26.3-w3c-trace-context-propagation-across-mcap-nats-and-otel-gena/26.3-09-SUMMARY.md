## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] `grpc/agent.rs` telemetry + WebRTC relays: use outer `msg` not inner `m`**
- **Found during:** Task 1, sites #13 (:2183) and #14 (:2367).
- **Issue:** Plan's numbered-list instructions for sites 13/14 said to insert `if let Some(ref headers) = m.headers { ... }` using the inner match-arm binding `m`. The actual code shape is `let msg = tokio::select! { ... msg = sub.next() => match msg { Some(m) => m, ... } };` — the inner `m` is shadowed by the outer assignment into `msg`, so `m` is out of scope in the loop body where the extract belongs. Using `m.headers` would be a compile error. The plan's own hedged text ("if the outer variable the select assigns is named `msg` ... adjust accordingly") authorizes this adjustment.
- **Fix:** Used `msg.headers` (the outer assigned variable) at both sites. Matches the downstream `msg.payload` / `msg.subject` reads.
- **Files modified:** `crates/roz-server/src/grpc/agent.rs` (two of four insertions in this file).
- **Commit:** `75282f8`.
