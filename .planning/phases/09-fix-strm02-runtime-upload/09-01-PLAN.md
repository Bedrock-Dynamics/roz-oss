<objective>
Close the STRM-02 wiring gap: worker startup currently discards the EmbodimentRuntime and
passes None to upload_embodiment, leaving the server's embodiment_runtime DB column NULL
for all real workers. WatchCalibration gates on that column and returns NOT_FOUND
unconditionally. This plan changes main.rs to retain the full runtime and pass Some(&rt),
so the DB column is populated and WatchCalibration can serve real workers.

Purpose: STRM-02 requires the server to stream calibration overlays. That path is blocked
at gate 1 (embodiment_runtime column = NULL). This fix unblocks gate 1 for any worker
that loads a robot.toml with an embodiment/channels section.

Output:
- crates/roz-worker/src/main.rs — variable renamed, runtime retained, upload call updated
- crates/roz-worker/src/registration.rs — new unit test proving body shape
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Tampering | EmbodimentRuntime JSON body | accept | The worker is a trusted internal process operating with an API key; no external path injects the runtime JSON. serde_json deserialization on the server side already rejects malformed payloads. |
| T-09-02 | Information Disclosure | embodiment_runtime DB column now populated | accept | The column is already guarded by bearer auth on the PUT endpoint (Phase 05); populating it does not widen the exposure surface — reads require the same auth. |
| T-09-03| Denial of Service | Large EmbodimentRuntime serialized payload | accept | upload_embodiment already has a 10-second reqwest timeout (Phase 05 Task 1). The runtime struct is bounded by the robot.toml schema; no unbounded fields are added by this fix. |
| T-09-04 | Spoofing | Worker impersonation (sends fake runtime) | accept | API key authentication on PUT /v1/hosts/{id}/embodiment is already enforced; this change does not alter the auth model. Host-ID in the path is UUID from the registration response — same trust level as before. |
